HTTPServer — How It Works

A multithreaded HTTP/1.1 server in header-only C++20 (Winsock, port 8080).

2 · Request Lifecycle

What happens from main() startup through serving a single request.
1
Winsock init & listen socket — main.cpp
WSAStartup → socket() (wrapped in SocketGuard RAII) → bind() to port 8080 → listen(). Each step is error-checked.
2
Create the worker pool & router — main.cpp
ThreadPool pool(4) spins up 4 std::jthread workers that block on a condition variable; Router router holds the registered routes.
3
Accept loop — main.cpp
accept() blocks until a client connects, then the raw socket is handed to pool.enqueue(...) as a handleClient task and ownership is released from the guard.
4
A worker picks up the connection — ThreadPool.h → ClientHandler.h
A free worker thread dequeues the task and runs handleClient(), which loops to support HTTP keep-alive.
5
Read the raw request — readRequest()
recv() loops until \r\n\r\n (end of headers), then reads exactly Content-Length more bytes for the body. Proper framing — no fixed-size assumptions.
6
Parse it into a struct — HttpRequest.h → UrlUtils.h
parseRequest() extracts method/path/version + headers + body, splits the query string on ?, and URL-decodes the path (%XX and +).
7
Route the request — Router.h
route() matches method + path segments against registered handlers (including :param patterns). No match falls through to static file serving.
8
Produce a response — handler or FileServer.h
A matched handler builds an HttpResponse; otherwise serveFile() reads from ../public in binary mode and picks a Content-Type from the extension.
9
Serialize, log, send — HttpResponse.h + Logger.h
buildResponse() assembles the status line + headers + body into a string; log() prints a timestamped line via std::format; send() writes it back.
10
Keep-alive or close
If the request's Connection header isn't close, the loop goes back to step 5 on the same socket; otherwise the socket is closed and the worker returns to the pool.

3 · File Reference

main.cpp
Entry point. Sets up Winsock, the listening socket, the thread pool and router, then runs the accept loop.
uses: ClientHandler, ThreadPool, Router
ClientHandler.h
readRequest() frames bytes off the socket; handleClient() drives the per-connection request/response loop with keep-alive.
uses: Socket, HttpRequest, HttpResponse, Logger, Router
ThreadPool.h
Fixed pool of std::jthread workers fed by a mutex-guarded task queue, with cooperative cancellation via stop tokens.
uses: std only
Socket.h
SocketGuard — RAII wrapper that closesocket()s on destruction; non-copyable so ownership is explicit.
uses: winsock2
Router.h
Maps method + path-segment patterns to handlers, supports :param captures, and falls back to static files.
uses: HttpRequest, HttpResponse, FileServer
HttpRequest.h
parseRequest() turns the raw bytes into a struct: method, path, version, headers, body, query params.
uses: UrlUtils
HttpResponse.h
The response struct plus buildResponse(), which serializes status line + headers + body into a wire string.
uses: std only
FileServer.h
serveFile() reads a file from ../public in binary mode and assigns a MIME type by extension (404 if missing).
uses: HttpResponse
UrlUtils.h
urlDecode() for %XX escapes and parseQueryString() for k=v&... pairs (with + → space).
uses: std only
Logger.h
log() — one timestamped line per request (method, path, status) built with C++20 std::format + <chrono>.
uses: std only
